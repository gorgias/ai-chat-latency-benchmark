Methodology

How latency was measured
🎯

What's measured

Wall-clock time from postMessage("sendMessage") dispatched to Siena's iframe until a new bot message appears in the GET /v1/live_chat/message poll — matching user-perceived latency exactly.

🔌

Widget reverse-engineered

Siena's live widget (chat.siena.cx) uses api-prod.siena.cx. The benchmark drives the widget's own JavaScript code path — no mocking, no shortcuts.

🔄

Polling cadence

Responses polled every 500 ms. Maximum wait per turn: 30 s. The ±500 ms polling granularity is included in all reported latencies.

📍

Context

Single session, 10 sequential shopping questions about the Mesa Loop 30oz. Siena's backend connects to the merchant's Gorgias helpdesk for escalation.
